# 删除实验 · 盲测裁判

> 生成日期 2026-07-07｜引擎 达人雷达 Creator Radar｜品牌配置 insta360
> 池子 `data/pool/creator_pool.jsonl`（1106 频道，26 正例 = 20 中腰部 + 6 巨星）｜本地 bge-m3 embedding｜串行跑，零 API

## 一、方法

每个变体 = 一份改过的 config 副本，只动打分旋钮，不动 `src/` 一行代码（`backtest.py` 吃 `--config` 就够）。全池零样本重跑，正例做隐藏标准答案。所有变体与本轮对照组 **fresh-baseline** 比，不与老基线（2026-07-06-pilot）比，以吸收 ollama embedding 的浮点漂移与池子 +21 频道的影响。

信号三路（semantic / sweet_spot / platform_markers）代码里不自动归一，删某路后按原比例把余下路重归一化到和 = 1。主题六个在代码里已被权重和除掉（等价自动归一），删某主题后仍显式把余下主题重归一化到和 = 1，让 config 自解释。

**判读规则**（对比 fresh-baseline）：

- 召回掉 ≥ 2 个百分点，或 正例中位百分位恶化 ≥ 2pp（中位数变大）→ **承重墙**
- 三指标变化全部 < 0.5pp → **白重量候选**
- 介于两者之间 → **边缘**（列证据，不下死判）

中位百分位越低越好（正例排得越靠前）。漂移噪声约 ±0.1pp 量级，0.1–0.2pp 的抖动一律读作噪声不读作信号。

## 四、每零件判决

### 承重墙（删了显著变差，必须留）

**sweet_spot 甜点函数：全系统最重的一堵墙**
- 删掉后前 5% 召回 23% 掉到 4%（-19.3pp），正例中位 32.4% 恶化到 53.7%（+21.3pp），中腰部中位 51.2%，基本等于随机洗牌（随机 = 50%）。
- 判读：「早期下注」不是叙事修辞，是引擎本体。池子本来就全是摩托/骑行频道，人人都「像」，没有规模先验，语义相似度分不开人。

**platform_markers 平台标记：第一页的聚光灯**
- 删掉后前 5% 召回 23% 掉到 12%（-11.6pp），但中位反而略好（-1.6 / -3.2pp）。
- 判读：它的全部价值集中在榜单最顶端，把甜点加语义筛出的人推进前 5%。产品承诺是「翻第一页」，前 5% 召回就是钱指标，必须留。留它的代价是中段排名略有牺牲，属于设计上可接受的取舍。
- 交互效应（答辩弹药）：markers 只在 sweet_spot 在场时兑现价值。semantic-only 前 5% 召回 4%，加上 markers（即 no-sweet）还是 4%；而在 semantic+sweet 的底子上加 markers，前 5% 从 12% 翻倍到 23%。三路信号是三脚架不是加法，缺一条腿另两条补不回来。

**semantic 语义（联合判读，矩阵里没有 no-semantic 变体）**
- semantic-only 中位 54.2%，约等于随机。语义单独扛不动这个池子，原因同上，同质池里相似度饱和。
- 但它不是白重量：no-markers（semantic+sweet）中位 30.8% 远好于随机，且下方主题逐删显示语义内部结构承重。本轮未单测 no-semantic，列为局限。

**pov_native 第一视角（权重 0.25，六主题里最重的墙）**
- 前 5% 召回 all -7.7pp、mid -10.0pp，中腰部中位 +2.0pp。
- 判读：与逆向工程的判断一致，第一视角是影石选人的核心信号，权重给得最重，盲测证明给对了。

**authentic_vlog 真实 vlog**
- 前 5% 召回 all -7.7pp、mid -10.0pp，前 10% 也掉（-3.8 / -5.0pp），中位反而改善（mid -4.0pp）。
- 判读：与 markers 同型，顶部聚光灯。删掉它省不出简洁，只丢第一页命中。

**journey_narrative 长途叙事（过线但幅度最小的墙）**
- 前 5% 不动，前 10% 两套各掉一个频道（-3.8 / -5.0pp），全体中位 +1.4pp。
- 判读：按红线判承重，但证据幅度 = 1 个频道。它守的是前 10% 这一段，不守第一页。

**gear_native 器材原生（软墙，单指标过线）**
- 六格召回全部纹丝不动，但中腰部中位恶化 +2.7pp（红线 2.0）。
- 判读：不影响第一页，托的是中段排名。判承重，注明证据来自单一指标；如果未来只看前 5%/10% 召回，可复议降级为边缘。

### 白重量候选（删了几乎无损）

- 无。没有任何零件落进「六项变化全部 <0.5pp」的静默带。这份 config 里没有纯装饰零件。

### 边缘（有变化但未过红线，列证据不下死判）

**adventure_bold 冒险大胆（唯一的真删除候选）**
- 六格召回与对照组逐格相同（全部 ±0.0），两套中位还各改善约 1pp（-1.0 / -1.2）。
- 判读：按规则落边缘带（变化 ≥0.5pp），但方向全部有利。删掉它系统不变差，甚至略顺。本轮唯一「删了不疼」的零件。

**vertical_craft 垂类技艺**
- 召回全格不动，中腰部中位 +1.4pp（高于噪声 ±0.1，低于红线 2.0）。
- 判读：接近可删，但中腰部付了小代价。权重只有 0.10，留的成本也低，先留着观察。

## 五、删删删结论

**一句话：9 种删法，7 种疼，1 种不疼反而略顺，1 种存疑。这套 config 已经非常接近「删到再删一个就跑不动」的正好状态。**

可以删的：

1. **adventure_bold 主题**。召回六格逐格不变，中位略好，是实操意义上的白重量。建议从 `config/insta360.json` 删除，六主题变五主题，并把「用盲测裁判删掉了一个主题」写进报名材料的迭代记录，删除哲学的活演示。此项等 Max 拍板后执行，本轮实验未动现役 config。

不能删的（按承重从重到轻）：

1. **sweet_spot** 是龙骨，删掉系统坍成随机。
2. **platform_markers、pov_native、authentic_vlog** 是第一页的三盏聚光灯，删任何一盏前 5% 召回都腰斩或重伤。
3. **journey_narrative、gear_native** 是细墙，各守一段（前 10% / 中段），过线但幅度小。

挂起观察的：

1. **vertical_craft**，接近可删，中腰部小代价，池子扩大或正例增多后复测再判。

方法论收获（可进答辩）：

1. 三路信号是三脚架不是加法，markers 的价值只在 sweet_spot 在场时兑现。交互效应只有删除实验暴露得出来，单看权重大小看不出来。
2. 裁判分辨率受正例数限制，召回最小步长 1 个频道（all 3.8pp / mid 5.0pp），中位数才是灰区显微镜。v2 扩正例集可提高分辨率。
3. 局限：本轮未测 no-semantic（语义是躯干，主题逐删已覆盖其内部结构）；边缘判定基于单次运行，ollama 漂移 ±0.1pp 不影响本轮任何判决（最小有效信号 0.9pp）。

下一步建议：

1. Max 拍板后：从现役 config 删 adventure_bold，重跑一次盲测确认逐格不变，写进迭代记录。
2. vertical_craft 标记观察，v2 复测。
3. 把本表连同判读规则收进附件母版（架构设计书）的验证章节，作为「每个零件都过过堂」的证据。
